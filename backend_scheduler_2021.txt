OS 및 일반적인 지식 --- 운영체제 : CPU, IO등을 관리하여 동시에 여러 App을 작동할 수 있도록 하는 환경 (21.09.12)
                --- Loading : CPU가 비휘발성의 ROM 데이터를 읽는다. 전원이 켜지면 부트로더를 RAM으로 넘긴다.

프로세스 관리 --------- 프로세스 : 메인 메모리에 할당되어 실행중인 상태인 프로그램,(21.09.13)
           --------- 큐 : 프로세스의 순서를 대기하는 곳, 이 순서를 정하는 과정을 스케줄링이라고 함

스레드와 동시성 -------- 스레드 : 하나의 프로세스 안에서의 흐름 단위라고 생각하면 된다. 경량의 CPU 사용하는 기본 단위이다(21.09.14)
                                Thread ID, PCC, register, stack으로 구성
            -------- 멀티스레딩 : 하나의 프로세스 안에서 둘 이상의 스레드가 동시작업을 수행하는 것, 자신이 속한 프로세스의 메모리를 공유한다.
            -------- 동시성 : 여기서 동시성이란 흔히 생각하는 동시가 아닌 멀티테스킹을 위해 여러개의 스레드가 빠르게 번갈아가면서 한번에 실행되는 것처럼 보이는 것을 말한다.
            -------- 병렬성 : 멀티 코에서 멀티 스레드를 동작하는 방식, 한 개 이상의 스레드를 포함하는 각 코어들이 동시에 실행되는 성질

기본적인 터미널 명령어 --- 해당 명령어는 실제 사용하면서 익혀야 할 듯 싶음. 기본적으로 암기가 필요한 부분이라고 생각되며, 이해가 되지 않는 부분도 존재(21.09.15)

메모리 관리 ------ 메모리 관리 : 프로세스를 위해 메모리를 할당하고 제거, 보호하는 활동. 이를 관리하는 관리자가 존재하고 이 관리자는 여러 정책(policy)를 따라 메모리를 관리(21.09.16)
            --- 정책(Policy) 종류 
                --- Fetch Policy : 적재 정책으로 디스크로부터 메모리로 프로세스를 언제 가져올지를 결정하는 것(요구가 있을때마다, 예상하여서 등 방법)
                --- Placement Policy : 어느 위치에 프로세르를 적재할 것인가를 결정하는 것.
                --- Replacement Policy : 메모리가 충분하지 않을 때 현재 메모리에 적재된 프로레스 중 어떠한 프로세스를 제거할 것인가를 결정.

프로세스간 통신(IPC : Inter Process Communication) (21.09.17)
        ------ 데이터를 주고 받는 과정을 가능하도록 하기 위해서 프로세스들이 동시에 접근가능한 공유하는 메모리가 필요
        ------ pipe, socket, share memory 등의 방법을 사용

                
POSIX(Portable Operating System Interface) (21.09.18) / 블로그 정리하면서 조금 더 정리하여 찾아보자
        ------ 한 운영체제에서 개발한 프로그램을 다른 운영체제에서도 쉽게 돌아가도록 하는 표준

네트워크 기초
        ------ 네트워크 : 동일한 프로토콜을 사용하는 디바이스들의 집합, 네트워크에 연결하여 서비스를 이용하거나 제공하는 것들을 디바이스라고 한다.
        ------ 네트워킹 : 디바이스 간의 데이터 전송, 
        ------ 네트워크 통신 : 주소(ip, 포트), 데이터
        ------ OSI 7계층 : 7층의 기본 개념을 이해하되, 궁극적으로는 최상위단 네트워크와 관련된 프로토콜에 대해서 이해
                ------ 어플리케이션 층(7) : End 유저와 컴퓨터가 통신하는 곳(일반 응용 서비스)
                ------ 트랜스포트 층(4) : 데이터 전송 서비스 제공(어플 층에서 전달한 데이터를 목적지 프로세스까지 전달하는 역할)
                ------ 네트워크 층(3) : 두 원격 시스템간 연결성과 경로 선택 제공

        인터넷 어플리케이션 프로토콜 : 어플 레이어에서 표준으로 지정된 주요 프로토콜
                ------ telnet : 원격 컴퓨터 접근 프로토콜
